Exception handling overview

Exception handling overview

We do not live in an error-free world.
Anticipate the possible errors in your code and deal with them in timely manner.
Handle the exceptional circumstances.

What is exception

Exception is an event, occuring during the execution of a program (run-time errors), that disturbs the normal flow of the program
Exception !== Syntactic Error
Syntactic errors can be detected during the compilation phase!
Classical exception example: "division by zero".

How JavaScript handles errors?

Instances of Error objects are thrown as exceptions when runtime errors occur.
The Error objects may also serve as base objects for user‐defined exception classes (explained deeply in next slides)

            var myErr = new Error([message])
message: [optional] Human-readable description of the error.

example


            function validateEmail(email) {
                if( email.match(/[\w.-]@\w+\.\w+/) ){
                    return email
                }else{
                    throw new Error(`invalid email: ${email}`);
                }
            }

            var validEmail = validateEmail('name@domaincom');

Exception handling with try-catch/finally blocks

If you do not want your program just to crash, but to handle possible exceptions gracefully:
try...catch
try...finally
try...catch...finally

try...catch


            "use strict";
            try{
                x = 2;
                console.log(`x=${x}`);
            }catch(err){
                console.error(`Error: ${err.message}`);
            }
if error occurs in try block, the execution is stopped and transferred to catch block.
If there are no errors => the catch block is not executed.

watch the scope


            try{
                x + 2;
            }catch(err){
                console.error(`Error: ${err.message}`);
                console.dir(err); // ok
            }

            console.dir(err); // ReferenceError: err is not defined
err - an instance of Error object. It's local scoped to catch block.

try...catch example


            var email = 'namedomain.com';
            try {
                const validatedEmail = validateEmail(email);
                if(validatedEmail instanceof Error) {
                    console.error(`Anticipated: ${validatedEmail.message}`);
                } else {
                    console.log(`Valid email: ${validatedEmail}`);
                }
            } catch(err) {
                console.error(`Unanticipated: ${err.message}`);
            }

            function validateEmail(email) {
                return email.match(/[\w.-]@\w+\.\w+/) ?
                    email :
                    new Error(`invalid email: ${email}`);
            }

the finally block

The statements in finally block are executed after the try block and catch clause(s) execute.
finally block executes whether or not an exception was thrown.
It's the right place to clean resources used in try-catch.

the finally block


            try{
                console.log(`Try 1`);
                x + 2;
            }finally{
                console.log("Finally 1");
            }

            console.log("But it's OK now");

Throwing exceptions

You can “throw” (or "raise") errors, as well.

                throw expression;
            

                throw 'Error 1';
                console.log("Post-throw action"); //never
the statements after throw won't be executed.

example


            function divide(x, y){
                try{
                    if( y === 0 ){
                        throw("I would not divide by zero!");
                    }else{
                        return x/y;
                    }
                }catch(e){
                    console.dir( e );
                    return undefined;
                }
            }

            var res = divide(5, 0);
            console.log(`res: ${res}`);

example


            "use strict";
            var testNode = document.getElementById("test");
            function addClass(element, className){
                try{
                    if( className == undefined){
                        throw("missing class name!")
                    }
                    element.className += " " + className;
                }catch(e){
                    console.log(e);
                }
            }

            addClass(testNode, "box");
            // note: This is just an example!
            // Element.classList.add() is better to be used.

Custom exceptions

ES5 way


            function DivideByZero(message) {
                this.name = "DivideByZero Error";
                this.message = message || "Can not divide by negative value";
                this.stack = Error().stack;
            };
            DivideByZero.prototype = new Error;

            function divide(x, y){
                try{
                    if( y === 0 ){
                        throw( new DivideByZero() );
                    }else{
                        return x/y;
                    }
                }catch(e){
                    console.log( e.name );
                    console.log( e.stack );
                    return undefined; // or whatever we want
                }
            }

            console.log( `5/0= ${divide(5, 0)}` );

ES6 way


            'use strict';

            class DivideByZero extends Error {
              constructor(msg) {
                super(msg);
                this.name = 'DivideByZero Error';
              }
            }

            function divide(x, y){
                try{
                    if( y === 0 ){
                        throw( new DivideByZero() );
                    }else{
                        return x/y;
                    }
                }catch(e){
                    console.log( e.name );
                    console.log( e.stack );
                    return undefined; // or whatever we want
                }
            }

            console.log( `5/0= ${divide(5, 0)}` );

window.onerror - the global error event handler

Syntax


            window.onerror = function(message, source, lineno, colno, error) { ... }
message: error message (string).
source: URL of the script where the error was raised (string)
lineno: Line number where error was raised (number)
colno: Column number for the line where the error occurred (number)
error: Error Object (object)
